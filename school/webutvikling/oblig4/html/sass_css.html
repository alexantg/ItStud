SASS og CSS

23.10.2019

Denne artikkelen skal forklare hvordan SASS fungerer i forhold til ordninær CSS. SASS er i all hovedsak et utvidelsesspråk til orndinær CSS. Den tar i bruk liknende programmeringssyntax for å kunne oppnå dynamisk CSS. SASS-filer kan ikke benyttes direkte i HTML-en og må pakkes sammen før det brukes.

I SASS kan du istendefor å skrive flere spesifikke selectorer bruke nestede selectorer. Her kan man skrive hvilket element man vil stilsette innenfor foreldre-elementet. Dette blir ikke umiddelbart mindre kode, men er mer overskiktlig og lettere å holde styr på når man skriver my CSS.

Du kan også definere variabler som kan holde på en rekke ting. En variabel kan for eksempel holde på en fargekode. Disse variablene kan du bruke videre i stilarket. Variablene kan selvføglelig holde på mer enn bare fargekoder også.

Man kan i SASS bruke små funksjoner som kalles mixins. Disse kan brukes til å generere CSS-kode, og kan spare oss fra å skrive mange linjer med vanlig CSS. Funksjonene kan brukes om igjen, dette er da endel av denne programmeringssyntaxen som er nevnt ovenfor. For å oppnå struktur i SASS, deler man ofte opp filene i ulike deler. Deretter referer man/ implementerer man disse filene i en hovedfil, som er den som skal bli pakket.

Merk, SASS-filene som er lagt ved mappen på dette nettstedet er eksempelet fra forelesning. Jeg har foreløpig ikke hatt mulighet til å løse oppgave 4 på en ordentlig måte, pga sykdom.
